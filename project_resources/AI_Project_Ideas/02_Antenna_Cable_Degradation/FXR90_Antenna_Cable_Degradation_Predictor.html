01 / Executive Summary
Project Summary

Predict RF path degradation before tag reads fail

Antennas and RF cables are exposed to vibration, movement, dust, physical damage, connector looseness, moisture, and installation drift. A reader may still be online while one antenna path slowly becomes weak. This project detects that early by comparing per-antenna read volume, RSSI, phase, channel behavior, connection state, and cable compensation settings over time.

Read Drop Detect falling tag reads per antenna compared with baseline.
RF Quality Track RSSI, phase consistency, channel spread, and read count.
Cable Risk Use cable length and loss compensation as context.
Disconnect Identify hard disconnects and intermittent antenna state changes.

Business Problem

  • One weak antenna can reduce RFID coverage while the reader still appears healthy.
  • Loose connectors and cable wear often show gradual RF decline before total failure.
  • Manual inspection is slow when many dock doors, portals, or read zones exist.
  • Operations need per-antenna alerts with clear field-service action.

AI Outcome

  • Health score for each antenna/read point.
  • Predicted issue type: disconnect, cable loss, misalignment, obstruction, or configuration mismatch.
  • Confidence and trend explanation.
  • Recommended inspection or corrective action.

Showcase Statement

The project turns normal FXR90 tag-read telemetry into an AI maintenance assistant that finds weak antennas and degrading RF cables before inventory accuracy or portal performance drops.

Important Measurement Note

RSSI and read counts are affected by tag population, product mix, antenna placement, reader mode, transmit power, and environment. The model should compare each antenna against its own historical baseline and current operating configuration, not a single global threshold.

Failure Modes
03 / What We Predict
Prediction Targets

Classify the most likely antenna or cable problem

The system should not only say “bad antenna.” It should estimate the most likely issue type so the technician knows where to start.

Predicted Failure Classes

  • Hard disconnect: antenna status changes to disconnected or unknown.
  • Intermittent cable/connector: connection flips or RF strength drops in bursts.
  • Gradual cable loss: RSSI and read rate decline slowly over days or weeks.
  • Antenna misalignment: read zone becomes weaker but connection remains stable.
  • Obstruction/environment change: one zone degrades after layout or material change.
  • Configuration mismatch: read drop caused by changed mode, disabled port, or transmit power.

Example Alert

Antenna 3 degradation risk is High.

Read volume dropped 42% versus the 14-day baseline, median RSSI is 6.8 dB weaker, and transmit power/configuration did not change. The most likely cause is cable loss, loose connector, or antenna misalignment.

Action: inspect Antenna 3 cable path, tighten connectors, check antenna mounting angle, then rerun validation scan.

Symptom Pattern Likely Root Cause Evidence Used
Antenna state reports disconnected. Physical disconnect, failed cable, failed antenna, failed port. radio_control.antennas, get_status.antennas, error/warning messages.
RSSI declines while read volume declines. Cable degradation, connector looseness, antenna damage, antenna moved away. peakRssi, reads, numTagReadsPerAntenna.
Read volume drops but RSSI remains normal. Traffic/tag population change, obstruction, workflow change, zone usage change. Reads by antenna, expected operating schedule, neighboring antenna comparison.
RSSI is weak after known cable setting change. Incorrect cable loss compensation or cable metadata mismatch. get_cableLossCompensation, set_cableLossCompensation audit, RSSI baseline.
All antennas degrade together. Reader config, RF environment, reader power, tag population, or site-wide interference. get_mode, transmit power, environment profile, radio state, warnings.

Best MVP Target

Start with per-antenna degradation detection for connected antennas: read-rate drop, RSSI drop, and no matching configuration change. This gives high operational value and avoids overcomplicating the first release.

Prediction Design
04 / Health Score
AI Logic

Score each antenna using trend, quality, and configuration context

A useful model needs to avoid false alarms caused by normal traffic changes. The score should blend antenna connection status, read-volume trend, RF signal trend, variance, and configuration checks.

Score Components

30%
Read-rate drop

RSSI degradation

RF variance

10%
AntennaDegradationScore =
0.30 * connection_risk
+ 0.25 * read_rate_drop_risk
+ 0.25 * rssi_drop_risk
+ 0.10 * rf_variance_risk
+ 0.10 * config_mismatch_risk

Risk Level:
0-39 Low | 40-64 Medium
65-84 High | 85-100 Critical

Baseline Features

  • Median RSSI per antenna per hour.
  • Tag reads per antenna per operating window.
  • Unique tag count per antenna.
  • RSSI percentiles and trend slope.

Comparison Features

  • Antenna vs its own 7/14/30 day baseline.
  • Antenna vs neighboring antennas.
  • Current RF config vs previous config.
  • Expected zone activity vs actual activity.

Diagnostic Features

  • Connection flips per day.
  • Warning and error message counts.
  • Radio connection health.
  • Cable length and loss metadata.
Model Stage Recommended Method Reason
MVP Rules plus per-antenna baseline anomaly detection. Works without labeled failure history and is easy to explain to operations.
Phase 2 Isolation Forest, robust z-score, or EWMA trend detection. Good for discovering unusual drops and slow degradation in time-series telemetry.
Phase 3 Supervised classifier using service records and confirmed root causes. Improves root-cause classification after maintenance feedback is collected.

Value and Conclusion
08 / Showcase Close
Business Value

Why this project is worth building

Antenna and cable issues directly affect RFID read accuracy. This project catches weak read paths before they become operational misses, and it gives technicians precise evidence instead of asking them to inspect the whole site.

Higher Read Accuracy Prevent blind spots and weak zones from growing unnoticed.
Faster Maintenance Tell technicians exactly which antenna or cable path to inspect.
Lower Downtime Fix degradation before read operations or portal workflows fail.

KPIs to Track

  • Missed read rate by portal or zone.
  • Number of antenna/cable incidents detected early.
  • Mean time to identify weak antenna path.
  • Mean time to repair antenna/cable issues.
  • False positive and false negative alert rate.
  • Post-maintenance RSSI/read-rate recovery.

Stakeholder Pitch

We can use FXR90 antenna status and tag-read telemetry to detect weak antennas and cable degradation before inventory accuracy suffers. The MVP is explainable, practical, and directly tied to operational reliability.

Recommended Next Step

Pilot on 3-5 readers with known antenna layouts. Capture at least two weeks of tag events and heartbeat status, build the per-antenna baseline, and validate alerts with field inspection results.
